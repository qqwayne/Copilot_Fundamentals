# Copilot Agents デモ

以下の3つのファイルを使って Copilot Agents デモをセットアップします。
`ISSUE.md` の内容を新しいリポジトリの Issue に貼り付けてください。

## このデモで示すこと
- Issue を Copilot に割り当ててエージェントタスクを開始する
- AgentHQ で進捗を監視する
- セッション中に新しい要件で作業を再指示する
- チームメンバーの作業として PR をレビューする

## デモファイル
- `ISSUE.md` は GitHub にコピー/貼り付けするための正確な Issue テキストを含みます
- `STEER.md` はエージェントが作業中に貼り付ける中間要件の変更を含みます

## ISSUE.md の目的と機能

### 目的
このIssueは、チケット振り分けポリシーに **優先度（Priority）システム** を追加することを目的としています。現状は**重大度（Severity）**のみで対応を決めているため、チームがチケットを一貫して処理できるよう優先度レベルを導入します。

### Copilot に依頼する機能
| 機能 | 内容 |
|------|------|
| 優先度レベルの定義 | P0〜P3 の4段階の優先度を追加し、それぞれの意味と基準を記載する |
| 重大度 → 優先度 マッピング | 既存の Low / Medium / High 重大度から P0〜P3 への明確な対応表を追加する |
| 振り分けチェックリスト | エンジニアが60秒で使える簡潔な振り分け手順チェックリストを追加する |

### 期待する成果物
- `README.md` に優先度セクション（定義付き）が追加されている
- 短い振り分けチェックリストが追加されている
- 簡潔でエンジニアがすぐに従える内容になっている

## クイックデモ手順
1. `ISSUE.md` の Title と Body をコピーして新しい GitHub Issue を作成する
2. Issue を Copilot に割り当ててエージェントタスクを開始する
3. AgentHQ を開いて進捗を監視する
4. 作業を再指示するために `STEER.md` をエージェントセッションに貼り付ける
5. PR の差分を明確さ、完全性、制約の観点でレビューする
6. PR が README.md を編集して優先度レベルと指示に従った例を追加していることを確認する

## チケット振り分けポリシー（現状）

今は**重大度（Severity）**だけで対応を決めています。

### 重大度レベル
- Low（低）: 小さな不便で簡単な回避策がある
- Medium（中）: 重要なユーザー影響があるが回避策が存在する
- High（高）: 主要なワークフローを阻害する、またはデータ損失を引き起こす

### 現在のルール
- 振り分けは毎日行う
- High 重大度を優先して対処する
- 現在、優先度ラベルや担当者、すぐに使える振り分けチェックリストは決まっていない
